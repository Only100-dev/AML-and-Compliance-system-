# Task 3.2c — Zod Query Parameter Validation for GET-only API Routes

## Summary
Added Zod query parameter validation to 6 GET-only API routes that were previously using raw `searchParams.get()` calls.

## Files Modified
1. `src/app/api/dashboard/route.ts` — `dashboardQuerySchema` (jurisdiction: string optional)
2. `src/app/api/compliance/route.ts` — `complianceQuerySchema` (jurisdiction: string optional)
3. `src/app/api/analytics/aggregate/route.ts` — `analyticsQuerySchema` (months: z.coerce.number int 1-60 optional)
4. `src/app/api/ubo-tree/route.ts` — `uboTreeQuerySchema` (entityId: string min 1, required)
5. `src/app/api/sanctions-exceptions/route.ts` — `sanctionsExceptionsQuerySchema` (status: string optional)
6. `src/app/api/audit-log/route.ts` — `auditLogQuerySchema` (action, userId, search: string optional; page, limit: z.coerce.number optional)

## Pattern Applied
- Import `z` from 'zod'
- Define schema before route handler
- After authGuard + rateLimit, parse `Object.fromEntries(searchParams)` with safeParse
- If `!parsed.success`, return 400 with `parsed.error.flatten().fieldErrors`
- Replace all `searchParams.get('xxx')` with `parsed.data.xxx`
- Use `z.coerce.number()` for numeric query params (auto string→number conversion)

## Lint Result
0 errors, 1 pre-existing warning (unrelated)
