### Architecture Decision: Derived State Pattern
- Chose NOT to use `useEffect` + `setState` for syncing API data to local state
- React Compiler rule `react-hooks/set-state-in-effect` prohibits direct setState in effects
- Instead, compute effective values directly from `profileData` + user edits
- User edits tracked separately via `nameEdit`, `phoneEdit`, `prefEdits` states
- `null`/`undefined` in edit states means "use API data", any other value means "user edited this"
- Boolean fields use `!== undefined` check (not `??`) to correctly handle `false` values
- This pattern avoids cascading renders and is recommended by React team
